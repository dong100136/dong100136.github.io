新的起点

发表于 2016-04-27

  我是来开坑的，从很久之前就很想写博客了，可以怎么都下不定决心（其实就是自己懒），博客在很久之前就已搭好了，就是没有开始动笔。拖延的毛病一直没有改过来。
  写博客主要是有两方面原因吧，一是希望锻炼自己的思维，强迫自己去思考，去表达，一直以来表达都是我的弱项，希望能借此机会进行锻炼。二是记录和总结我自己所学的东西，我是一个很有好奇心的人，对什么东西都很感兴趣，但是一直对自己很放纵，没有计划自己的知识架构，很多东西都是看一下，又不看了，没有深层次的积累，一会儿看一下nodejs，一会儿看一下go，一会儿看一下docker，诚然这不是一件好事，希望写博，可以让自己从烦躁中沉静下来，积累点东西。
  对于博客的内容的规划，主要是各种webApp，前端，后端的东西的整理总结，这是主要的，其次是对自己做的一些小项目，小作品进行反思总结和归档，最后也会有一些个人的总结，反思自己，大概就是这些。
  这个星期，老师组织了一次座谈会，两个多小时后，我唯一记得的是50小时理论，如果一个人能够一个星期内在一个领域花不少于50小时的有效时间，他就可以在这个领域横着走。这让我意识到一个问题，虽然我花了很多时间在计算机方面，然而只是一直在折腾这个东西，并没有正真的积累，效率是一个问题，同样是花费一个小时，正真有效的时间又有多少。很多时候，一天过去了，感觉自己好像什么都没有做，或者只做了一点点东西。所以我应该对我的时间进行有效的管理，把时间用起来。
  暂时想多的就这些吧，作为这个博客的首篇博文，这将是我走上大神之路的起点。

【前端恶补系列】position

发表于 2016-04-27

在html的布局中我们经常都要使用的position这个属性，下面是对position的一些总结：

常用的position属性及特点如下：

属性 特点
absolute 相对于于第一个非static属性的父元素进行定位，使用top，right等定位
relative 相对于父元素进行定位，但是只是视觉上，仍然占有位置，不脱离文档流，使用top，right等定位
fixed 相对于浏览器窗口进行定位，使用top，right等定位
static 默认使用的属性
inherit 继承父元素的定位属性

absolute 绝对定位

在absolute定位中，如果没有指定top，right等属性，它会自动选父元素进行定位，而不看父元素的定位属性。因此在不适用top等属性时，元素是没有脱离文档流的。

决定定位的元素可以通过z-index属性设计相互层叠关系，值得注意的是absolute元素之间的margin之间是不会发生重叠的

relative 相对定位

在相对定位中，无论是否使用top等属性，该元素都不会脱离文档流。

static 静态定位

默认定位方式，位于文档流中，top，right等属性不起作用
